BottleShip Weak-Model Browser Demo

This page visualizes BottleShip mechanics in a weak browser model. It is useful for learning the flow, but it is not the strong model and must not be treated as a secure implementation.

Unsafe for real secrets: weak-model visualization only.

The browser demo cannot enforce irreversible capsule destruction, rollback-resistant state, trusted key custody, or copy resistance. A user can inspect or modify the JavaScript, duplicate the archive state, or restore an earlier snapshot. The in-memory key capsules shown here are only a teaching aid.

The strong BottleShip model requires a trusted component that holds state outside user control, rejects rollback, and destroys capsules irreversibly. For the current non-browser simulator path, use the Go CLI at the repository root in cmd/bship. The prototype/ directory is reserved for future prototype notes and strong-model work only; it does not contain the current simulator implementation.

Weak demo vs. strong trusted-state model

Weak browser demo

  • Runs entirely in user-controlled JavaScript.
  • Stores simulated capsules only in browser memory.
  • Can illustrate threshold refusal and prune flow.
  • Cannot stop copying, rollback, or code tampering.

Strong trusted-state model

  • Uses a trusted component to hold capsule/state custody.
  • Requires rollback-resistant state across operations.
  • Must perform irreversible capsule destruction when pruning.
  • Is the path to evaluate for real security claims.

Keep using this page to explore the weak model only. For the current non-browser simulator path, use the Go CLI at the repository root in cmd/bship; prototype/ remains future work.
